jQuery事件模型

创建监听器

  • 示例
    $('img').bind('click', function(event){ alert('Hi there! ;'); });
  • 语法：
    • bind( eventType, data, hanlder )
    • 参数eventType
      • 标准事件名称，例如click、mouseover、keydown等。
      • 可以使用空格分隔的列表指定多个事件类型。
      • 可以在事件名称后添加圆点字符后缀来添加命名空间。
    • 参数data
      • 调用者提供的数据，给处理器函数使用。
      • 如果该参数省略，第二个参数就是处理器函数handler。
    • 参数handler
      • 传入Event实例
      • 设置函数上下文为冒泡阶段的当前元素。
    • bind( eventMap )
      • 参数eventMap
      • 一个javaScript对象，允许在一次调用中创建多个事件类型的处理器。
      • 对象属性名称指定事件类型。
      • 属性值指定处理器。

jQuery不支持捕获阶段。

命名空间

  • 为事件名称添加圆点分隔的后缀，来指定命名空间。
  • 可以使用多个后缀，将事件归入多个命名空间。
  • 可以根据事件分组，对事件进行批量操作。
  • 例如，可以编辑模式的页面元素事件加上.editMode后缀。

特定事件

  • focusin和focusout

    • 出现原因：从本质上看，获取焦点和失去焦点的事件，不会冒泡。
    • JQuery：这两个事件会冒泡。

  • one()

    • 事件处理器执行一次之后，就会被自动删除。
    • eventTypeName( listener )

      支持的事件：blur、change、click、dblclick、error、focus、focusin、focusout、keydown、keypress、keyup、load、mousedown、mouseenter、mouseleave、mousemove、mouseout、mouseover、mouseup、ready、resize、scroll、select、submit、unload

    • 注：使用这种便捷方式时，不能指定存储在event.data属性中的data值。

  • 删除事件处理器

    • unbind：为包装集中的所有元素删除指定处理器。
      • 格式：unbind( eventType, listener )、unbind( event )
      • 注：如果没指定参数，则删除元素上所有监听器。
      • 如果使用unbind( event )，则函数要设定为顶级函数，或者用变量保留引用，如果是匿名函数就无法引用了。
      • 使用命名空间：unbind(‘.editMode’);

Event实例

  • 使用独立于浏览器的JQuery.Event属性

  • 属性

    • altKey、ctrlKey、metaKey、shiftKey、：按钮是否按下，true或false。
    • pageX、pageY：鼠标相对页面原点的坐标。
    • screenX、screenY：鼠标相对屏幕原点的坐标。
    • currentTarget冒泡阶段的当前元素。他和事件处理器中函数上下文是同一个对象。
    • target：触发事件的元素。
    • data：创建时间处理器时，作为第二个参数传入bind()方法。
    • result：从前面事件处理器返回的最近的非undefined的值。
    • timestamp：jQuery.Event实例创建的时间戳，以毫秒为单位。
    • type：为所有的事件指定触发的事件类型。
    • which：对于键盘事件，指定出发时间的按键的数字编码，对于鼠标事件，指定按下的是哪个按钮，1左2中3右。

  • 方法

    • preventDefault()：阻止任意默认的语义动作发生。比如表单提交、链接重定向、复选框状态的改变等。
    • stopPropagation()：停止事件冒泡。当前目标元素上附加的事件不受影响。不仅支持浏览器事件，还支持自定义事件。
    • stopImmediatePropagation()：停止所有时间的进一步传播，包括附加在当前目标元素上的事件。
    • isDefaultPrevented()：判断此实例上是否调用了preventDefault()方法。
    • isPropagationStopped()：判断此实例上是否调用了 stopPropagation()方法。
    • isImmediatePropagationStopped()：判断此实例上是否调用了stopImmediatePropagation()方法。

  • keyCode问题

    • 使用keyCode属性在不同浏览器下处理非字母字符是不可靠的。
    • 可以在keypress中使用which属性获得可靠的字符代码。
    • 在keyup和keydown中，使用which只能获得不区分大小写的按键代码，但是可以通过检查shiftKey来确定大小写。（怎么检查CapsLock？）
    • 其他：如果想停止事件的传播，而且取消他的默认行为，那可以让监听器函数返回false。

预先管理事件处理器

  • live()：可以预先为那些还不存在的元素创建事件处理器。
  • 格式：live( eventType, data, listener )
  • 与bind()的差异
    • live()触发匹配元素上的事件时，不会继续传播事件。所以，依赖传播和传播顺序时，不应该用live()
    • live()只能用于选择器，不能用于衍生出来的包装集。
    • 使用die()删除live事件处理器。

触发事件处理器

  • 将事件处理器作为函数调用，不会触发语义动作，也不会导致冒泡。
  • trigger( eventType, data )
    • 可以在eventType后面追加感叹号，来阻止命名空间事件的触发。
    • 会在DOM层次结构中传播，还会执行语义动作。
    • Event实例上的target属性会被设置为与处理器绑定的匹配集元素的引用。
    • 可以通过Event实例上的stopPropagation()方法，或调用的处理器返回false来停止冒泡。
    • 注：传入trigger()和创建处理器时传入的data参数的区别。
  • triggerHandler( eventType, data )
    • 不会冒泡、不会执行语义动作、不会触发live事件

  • 其他

    • toggle()：循环调用的监听器。

      • toggle( listener1, listener2, listenerN… )

    • hover()：鼠标进入或离开元素覆盖区域时候触发。

      • hover( enterHanlder, leaveHandler )
      • hover( handler )
