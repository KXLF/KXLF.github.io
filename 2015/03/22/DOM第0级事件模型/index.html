DOM第0级事件模型

DOM第0级规范是在DOM Level 1提出之前的各浏览器厂商已经比较通用的规范。
它还有以下几个其他的名字：

  • 网景事件模型：Netscape Event Model
  • 基本事件模型：Basic Event Model
  • 浏览器事件模型：Browser Event Model
  • DOM 第0级事件模型：DOM Level 0 Event Model

如何使用

将一个函数实例的引用赋值给DOM元素的属性。

  • 在标记特性中声明
    <input type="text" click="showIt('duang!');">
  • 将事件处理函数的主体作为特性值嵌入到DOM元素的HTML标记中
    <input type="text" click="alert('duang! duang! duang!'); return false;">
  • 用脚本控制声明
    $("#test1").onclick = function(event){
  showIt("duang! duang!");
}

缺点

  • 某个元素上的某种事件类型只能注册一个事件。

    当然可以通过包装多个函数、实现观察者模式、闭包技巧等方法来实现，但会增加复杂度。

Event实例

  • 获取Event实例
    • 标准浏览器：将Event实例作为第一个参数传入处理器。
    • IE浏览器：将Event实例保存到event全局变量中。（也就是window的一个属性）
    • 兼容处理：
      if ( !event ) event = window.event；
  • 获取触发事件的元素
    • 标准浏览器：target属性
    • IE浏览器：srcElement属性
    • 兼容处理：
      var target = ( event.target ) ? event.target : event.srcElement;

事件冒泡

  • 描述：当触发一个DOM树元素的事件时，浏览器会检查这个元素是否创建了特定事件处理器，如果是，则触发，然后继续检查和触发父元素的特定事件，以此类推，直到DOM顶部。
  • 利用：它允许我们在任何层级的元素上创建处理器，以便处理发生在后代元素上的事件。

    冒泡中的target一直是目标元素不变，而不是current元素的父元素。也就是说，冒泡过程中，每一级的事件都是由目标元素触发。

  • 阻止冒泡：

    • 标准浏览器：调用Event实例的stopPropagation()
    • IE浏览器：设置Event实例的cancelBubble = true
    • 注：很多现代浏览器也支持cancelBubble，虽然它不是标准的一部分。

  • 取消默认语义动作执行

    • 例如：取消a标签重定向到href、form提交事件导致表单提交。
    • 方法：将事件处理器返回值设为false。
    • 利用：
      • 在表单提交事件处理器中，验证数据，数据有问题就返回false
      • 直接设置onsubmit=”return false;”，而使用ajax异步提交数据。
