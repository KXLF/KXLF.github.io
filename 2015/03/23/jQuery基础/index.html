jQuery基础

  • 核心是从HTML页面上获取元素，并对他们进行操作。
  • 首要目标是确保在所有主要浏览器下都能流畅的运行代码。
  • 支持插件的机制，而且不难编写。
  • $()是jQuery()函数的别名。

jQuery包装器（wrapper）

  • $()函数返回的特殊JavaScript对象，它包含与选择器相匹配的DOM元素数组.
  • 这些元素按照在文档中的顺序排列。这个对象拥有大量预定义方法，用于操作这些元素集合。
  • 简单来说，就是包装收集到元素，并添加了扩展功能。

jQuery作用链

  • 包装器方法执行完毕后，会返回相同的一组元素，来为另一个操作做准备。例如，隐藏CSS类名为item的全部标签，并为他们添加新的CSS类名removed：
    $(".item").hide().addClass("removed");

  • jQuery作用链可以无限延续，每个方法都作用在与最初选择器匹配的元素集合上。

  • 包装集也可以使用普通数组的方式访问，以下两行代码的效果是相同的：

    $("#someElement").html("Hello!");
$("#someElement")[0].innerHTML("Hello!");
  • 如果想匹配多个元素，并进行批量操作，jQuery可以直接对包装集进行操作，而普通数组只能遍历操作了。

实用函数

  • $()可以作为一些通用实用函数集的命名空间前缀。下面的两种用法效果相同，trim()是jQuery或者$命名空间下的一个函数：
    var trimmed = $.trim(someString);
var trimmed = jQuery.trim(someString);

文档就绪处理器

  • 整个页面主体加载完毕后再应用行为，传统方法是使用window实例的onload方法：
    window.onload = function(){
// 执行操作
}

  • 浏览器会延迟执行onload，直到DOM树创建完毕、所有资源也加载完毕的时候。这就导致用户需要等待很长时间加载完成，然后才能使用丰富的页面行为。

  • jQuery提供的更好的方法是在DOM树加载完毕后（而不包括所有外部资源），立即执行脚本：

    // 标准语法
jQuery(document).read(function(){
   // 执行操作
});

    // 简写形式
jQuery(function(){
   // 执行操作
});

  • 可以在一个页面中多次调用此函数，浏览器会按照他们的声明顺序依次调用。而window的onload机制只允许指定一个函数。

创建DOM元素

  • 示例：
    $(function(){  
   $("<p>Hi there!</p>").insertAfter("#followMe");      
});

扩展jQuery

  • 试图满足所有需求的库是不合理的，这可能会导致一大堆庞大的、臃肿的、包含很少可用功能的代码，这只会把事情搞砸。
  • jQuery只包含大部分页面开发需要的功能，放置于核心库中。
  • jQuery很容易通过附加功能来进行扩展。
    示例：
    // 禁用包装集中的元素
$.fn.disable = function(){
   return this.each(function(){
        if(this.disabled == null){
             this.disabled = true;
        }
   });
}
  • 在函数内部，this关键字指向将要操作的DOM元素集合。
  • 在传入each()的迭代器函数内部，this是对当前遍历的特定DOM元素的引用。
  • jQuery团队推荐了各种代码风格的完整列表，详细看这里。

jQuery与其他库共存

  • 遵循最佳实践原则，应避免引入大量的标示符而污染全局命名空间。
  • 标示符jQuery和其别名$是全局命名空间引入的唯一变量。
  • 如果$符号与其他库产生冲突，则调用jQuery.noConflict()即可将$给其他库使用了。（比如Prototype库）。

小结

jQuery()有如下用途：

  • 通过包装器方法选择和包装将要操作的DOM元素
  • 为全局实用函数提供命名空间
  • 从HTML标记创建DOM元素
  • 创建在DOM就绪后执行的代码
